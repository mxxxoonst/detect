# 编码器任务重定义与预训练设计

> 本文固化近几轮讨论的结论:**把编码器的主任务定义为"退化结构下的鲁棒表示重建"(自监督)**,
> 语义类型标注(CTA/PII)降级为下游任务。本文详细整理预训练阶段的输入构造、注噪算子、数据类型作用、
> 基座选型、分类头与损失函数。与"输入怎么造"的实现细节互补,见 [`span_segments_design.md`](./span_segments_design.md)。

---

## 一、任务重定义

### 1.1 主任务:退化结构鲁棒重建(self-supervised structural reconstruction)

> **给定带噪输入,重建/对齐干净结构**(去噪 + 边界/字段身份恢复 + 跨视图一致性)。

- **监督全部来自 synth 的 oracle**:边界在哪、某 token 属于哪个字段、注了什么噪、干净签名是什么——
  全是注噪时已知的事实,**零 regex、零 LLM、零人工标注争议**。这是科学上最干净的监督来源。
- **编码器/表示本身就是交付物和贡献**,不再是"达成某个外部语义标签"的手段。
- **编码器只覆盖形式层 + 结构层**(表面/结构腐蚀,信息在输入里、可去噪重建);**语义层**(黑话/同名异义,
  意义在输入外)交下游 DL/LLM 标注器(模块 B),见 §1.4 / §九。

### 1.2 为什么**不**把语义类型标注当主任务

| 质疑 | 说明 |
|---|---|
| 启发式标注不科学 | regex / name+profile 融合都太弱;CTA 早已是 Doduo/TURL/RACOON/LLM 的天下,**干净数据上的 CTA 不构成贡献** |
| LLM 标注 → 编码器沦为附庸 | 若用 LLM 当主任务监督,编码器天花板 = 老师 LLM,贡献退化成"更便宜的 LLM 近似" |
| 死结 | 只要把"语义类型预测"定为主任务,标签就**必须外求**:要么弱启发式(不科学)、要么 LLM(附庸),两条路都让编码器从主角沦为手段 |

**解法**:主任务换成一个**标签免费、且是 oracle、不依赖任何外部标注器**的对象——即 synth 的(干净 ⟷ 带噪)配对重建。
**"无标签"红利在预训练,不在下游**;下游 typing 是监督分类,按定义一定要标签,但因表示已预训练好 → few-shot。

### 1.3 richness 轴:腐蚀 × 格式 × 位置,**不是树深**

- 任务难度不在"嵌套深度"轴,而在"退化"轴。一张 depth-1 的 CSV,结构退化花样极多(列漂移/未转义分隔符/
  未闭合引号/表头坍塌/记录边界混入……),**depth-1 上依旧极其丰富**。
- 真正的 richness 空间:`{格式} × {结构位置} × {噪声算子} × {强度} × {层}`,组合上千量级,且每个都有 oracle 标签。
- **跨格式异构本身就是 table-QA 没有的结构难度**:把 JSON 树 / CSV 平表 / SQL 语句+元组 / xlsx 这些表面语法
  天差地别的东西,在带噪下统一成可比的记录结构——这是 format 轴的 richness,比单模态深嵌套更难。
- **不追深度,不去借 table-QA 的合并单元格/2D 布局**(那是空间推理,另一个任务,会稀释焦点)。

### 1.4 两模块划分:A(编码器)/ B(语义标注器)

三层噪声**按结构拆分到两个模块**(依据见 §九表):

- **模块 A = 编码器**:学**形式层 + 结构层**的结构鲁棒表示(encoder-only 自监督,oracle,零语义标签)。
- **模块 B = 语义类型标注器**:做**语义层**(黑话/同义/同名异义),用 DL/LLM——**三条路线**:
  TURL 式微调轻量头 / LLM zero-shot prompt / projection 对齐(见 §6.5 / §九)。
- **接口**:A 抽取出**干净结构化记录 + c_struct**(符号),B 消费之。两模块**任务解耦**(A 不需要
  语义标签),仅共享此符号接口 → 下游 B 的选择可延后。

## 二、预训练输入构造

### 2.0 两个解析器的分工(务必分清)

- **严格解析器**(ijson / 列一致 / scan_sql):**只在数据预处理阶段**用,从原始语料筛出 **tier1 干净种子**
  (`I_strict==1`)供 synth 扩充;**不参与带噪样本处理**。
- **容错解析器**(span 段流,见 `span_segments_design.md`):**观测输入构造器**,面向**带噪样本**,把带噪文件
  变成 **seg1(观测表面 + 试探归组)+ seg2(RAW)** 喂 A。
- **注意**:干净合成文件的 oracle 对齐(`clean_align`)由**精确 span 解析**(干净必然严格可解)得到,属
  **标签线**,与容错"观测线"分属两条、永不交叉(见 `noise_injection_spec.md`)。

### 2.1 span 级段流:`StructuredSegment | RawSegment`

容错解析升级为**按文件位置顺序产段**的生成器(实现见 `span_segments_design.md`):
- `StructuredSegment` = 恢复成结构的区域(保留表面 + 结构标注 + span + 锚点 + conf)。
- `RawSegment` = 救不回的坏区(原文切片 + span + 锚点 + reason)。

### 2.2 seg1:规范化恢复表面(canonicalized recovered surface)

把恢复出的结构重渲染成**保留语法骨架**的规范串:
1. **保留所有承载结构的标点** `{ } [ ] ( ) : , "` 和分隔符——形式层噪声活在这里,必须留给编码器看见;
2. **保留真实值,并行附类型标注**(不替换为类型槽):synth 语料的值由 LLM 生成、本身不敏感,且
   **值形(value-shape,如 `alice@gmail.com` 的形状)是字段定型的关键信号**——抹成 `<str>` 恰好丢掉 §6.2
   内容保留头要的东西。**PII 红线约束的是落盘持久化,不是编码器瞬态消费**(真实管线侧持久化仍按 CLAUDE.md
   脱敏,与此处编码器输入两回事)。值 token 上**并行**附 `⟨str⟩/⟨int⟩/⟨float⟩/⟨bool⟩/⟨null⟩` 作类型标注
   (标注,不替换);超长自由文本值可截断控长。
3. **归一非结构性表面噪声**(连续空白→单空格、去 BOM、键序规范化)。

例:`{"name":"Alice","age":30}` → 表面保留 `{"name":"Alice","age":30}`,值 token `"Alice"`/`30` 并行带
类型标注 `⟨str⟩`/`⟨int⟩`(对比当前 SchemaUnit 的 `[("name","str"),("age","int")]` 把括号/冒号/逗号**和值**全丢了)。

### 2.3 seg2:`[RAW]`

**救不回的坏区只能以原文进 seg2**(按定义不可能有"恢复表面")。结构理解恰恰被考在 seg1/seg2 的交界:
模型能不能把 RAW 块重新锚回结构(re-anchor)。

### 2.4 field 级 provenance 锚定接口(**锁 1**)

**provenance 有两个,务必区分(否则答案泄漏):**
- **观测 provenance(解析器在带噪输入上的段切分/试探归组,会错)= 输入**。可观测,但噪声下不可靠
  (列漂移/嵌套错位处尤甚)。可作输入。
- **oracle provenance(synth 已知的真实归属)= 标签**。注噪前干净结构知道每个值真正属于哪个字段——
  这是 field-binding 头(§6.1)的监督目标,**绝不作输入特征喂进去**。

**`H_fld` 池化的双时态(LUPI / privileged-info 教师强制):**
- **训练期**:用 **oracle provenance 当池化键**(teacher forcing)聚出 per-field `H_fld`,才能算 per-field 的
  `L_consist`/typing 损失;field-binding 头同时学着复现这个归属。
- **推理期**:oracle 不可得 → 改用**模型自己预测的 binding** 当池化键。
- 因此**同一批 token 从不"看见答案"**:编码器吃可观测表面、预测归属;oracle 只当(a)标签、(b)训练期池化
  拐杖,推理期撤掉。

编码器据上述池化出 **per-field `H_fld`**(**内部**用于 `L_consist` 与结构探针,**非 B 接口**;B 接口是抽取出的**符号结构化记录 + c_struct**,见 §1.4)。

### 2.5 输入序列拼装

```
[CLS] seg1(规范化恢复表面 + 结构标注) [SEP] [RAW] seg2(坏区原文) [SEP]
        ↑ 每 token 带 role/depth/node_type 标注 + 边集(均可从带噪表面观测)
```
- 输入侧结构标注**只含可观测信号**:`role`(open/close/delimiter/kv_sep/value_slot)、`depth`、`node_type`、
  解析器**试探性**归组——**不含 gold field_path**(那是标签,见 §2.4 / §6.1)。
- 单流拼接 + 每段一个"是否结构化"标记;不做重的跨通道机制。
- **防平凡复制**:干净区"观测归组==oracle"会令 field-binding 退化,故损失**向带噪位点加权**,并可对观测归组
  做 dropout,逼模型从表面而非现成归组学习。

## 三、注噪过程:算子 × 层次 × 类型

### 3.1 注噪形式化(均匀域随机化 + 结构标签保持界)

- 噪声算子 `o(x; θ)`:对干净样本 `x` 的每个可注位点以 `Bernoulli(p)` 决定是否施加,强度参数 `θ`。
- **分层均匀采样**(不匹配语料经验分布,假设理想均匀):
  先在层 `∈ {形式, 结构, 语义}` 上均匀 → 层内算子均匀 → 强度 `θ ∈ [0, θ_max]` 均匀。
- **结构标签保持界 `θ_max`**:注噪必须保持 oracle 结构标签可定义(干净结构仍是 ground truth,原则上可恢复)。
  越界的极端破坏(真 tier3)**只入 eval,不入 train**。

### 3.2 形式层算子(Form;解析器**解不了**)

| 算子 | 操作 | 参数 |
|---|---|---|
| 截断 truncation | 在单元的随机字节偏移处切断 | 切点 `θ ∈ (0,1)` |
| 未闭合引号/括号 unclosed | 删一个配对的闭合 token(`"` / `}` / `)`) | 位点采样 |
| 未转义分隔符 unescaped delimiter | 向某值内注入裸分隔符 | 注入数 |

> 字节/字符级随机腐蚀**不现实、不做**(用户已确认);形式层聚焦"结构性破坏",上述三类即可。

### 3.3 结构层算子(Structure;**可解析但 schema 漂移**)

| 算子 | 操作 |
|---|---|
| 列漂移 column drift | 增/删一个分隔符 → 列错位(over/under) |
| 嵌套错位 nesting misalign | 把键移到错误深度 / 仍能解析的括号错配 |
| 记录边界混入 record boundary mix | 值内插换行 / 合并两条记录 |
| 表头坍塌/缺失 header collapse | 删表头行 / 用某条数据冒充表头 |

### 3.4 语义层算子(Semantic;**可解析且结构正确,但意义不确定**)

**作用于字段名,保持 `y`(gold 类型)不变** —— 这是 synth 里唯一针对字段语义的算子,
也是批量造"难命名 + 已知类型"对的唯一手段(天然脏数据里 generic 名字 regex 直接 unknown,没金标)。

| 算子 | 例 |
|---|---|
| 缩写 abbreviation | `email → em / eml`,`id_card → idc` |
| 黑话/行业术语 jargon | 业务内部命名 |
| 同义替换 synonym | `phone → contact / tel` |
| 同名异义 homonym | 名为 `name` 实为公司名 |
| generic 化 | `col_0 / f1`(与结构层"表头缺失"重叠) |

> 连带约束:**值生成必须类型忠实**(email 字段填 email 形值),否则名字被砸后值通道反而骗模型、与标签矛盾。

### 3.5 注噪标签(noise-meta)与 oracle

注噪时记录:`{layer, operator, θ, 受影响位点/span, 干净结构签名, 干净字段身份, gold 类型 y}`。
这些是**主任务的免费监督**(边界/re-anchor/field-binding/一致性/c_struct 的 oracle),
也是下游 typing 的 `y` 来源(经 provenance 搬运到带噪输入)。

## 四、各数据类型的含义与作用

| 格式 | 结构形态 | 在任务里的作用 | 注噪适配 |
|---|---|---|---|
| **JSON** | 树:嵌套对象、arrays-of-objects、可选键、null 多态 | **深度/树结构的主要来源**(`UnionSchemaClusterer` 已在处理签名爆炸=结构变体丰富的铁证) | 嵌套错位、未闭合括号、截断 |
| **JSONL** | 行分隔记录 | 记录边界鲁棒性 | 行级截断、记录边界混入 |
| **CSV** | 平表(depth-1) | **形式层噪声最丰富、最现实**;表头语义 | 列漂移、未转义分隔符、未闭合引号、表头坍塌 |
| **TSV** | 平表,固定 `\t` | 同 CSV,分隔符确定 | 同 CSV |
| **SQL** | 语句 + 元组 | **`CREATE TABLE` 的列声明类型/ENUM/COMMENT = 字段语义类型的高质量先验**(本地可解析,当前 `_parse_create_columns` 只留列名、把类型尾巴丢了,应救回);语句边界鲁棒性 | 语句截断、未闭合括号 |
| **xlsx** | 二进制、结构化行 | **无容错侧**(二进制坏区没有有意义的 RAW);作干净/结构化锚点;表头语义 | 仅结构层(行级),不产 RawSegment |

> 注:FK 跨表拓扑**对本任务无用且实现昂贵**(粒度在 SchemaUnit 之上、不参与表面腐蚀、跨文件多悬空、
> 需全局符号表),**不纳入**。SQL 的结构红利在"列声明类型先验",不在 FK。

## 五、编码器基座选型:GraphCodeBERT vs CANINE(对比 / 消融)

**推理链**:抗表面腐蚀 = 任务本身 → 噪声**最先砸的是 tokenizer**;子词 tokenizer 在形式层噪声下会
**切分漂移**(注入逗号/删括号/截断 → 同段文本被切成完全不同子词),等于"模型没看到之前输入已被打乱";
字符级切分不级联、**编码无关**(适配 GBK/UTF-8 混编)、对任意保留值免 OOV。故本任务把**字符级、
tokenization-free、encoder-only** 作首选。

| 臂 | 粒度 | 强 | 弱 |
|---|---|---|---|
| **CANINE(主选)** | 字符,tokenization-free,内置下采样 | 抗字符腐蚀、编码无关、encoder-only、免 OOV | 无代码/结构预训练(靠 synth 自监督 + 轻量结构先验注入补) |
| **GraphCodeBERT(baseline 臂)** | 子词 + 结构预训练 | 结构先验强(token+边/AST) | **子词 tokenizer 恰是被形式层噪声击穿的一环**;干净码上预训、抗腐蚀未验 |

**结构先验注入**(不买代码模型"子词+AST"连带的脆 tokenizer):在 CANINE 字符基座上,`role/depth` 作附加
embedding、`same_field/kv_bind` 边作 attention bias。

**推荐**:**CANINE + 轻量结构先验 + synth 自监督预训练**为主;**GraphCodeBERT 作 baseline 臂**,专门
**量化"子词切分漂移在形式层噪声下掉多少点"**——这本身是可发表的消融结论。

## 六、分类头与损失函数

### 6.1 主任务自监督头(encoder-only 抽取式,**4 个核心头**)

**encoder-only 无生成 decoder → "重建"= 抽取式重分组**,boundary+field-binding+re-anchor 取代生成式 `L_recon`。核心 4 头:

| 头 | 任务 | 损失 | oracle 来源 |
|---|---|---|---|
| 边界 boundary | 逐 token 标记记录/字段边界(+`damaged`) | `L_bound` = token CE | 干净边界 |
| 字段绑定 field-binding | 每值 token 预测其字段身份 | `L_bind` = CE | 干净字段身份 |
| re-anchor | 每 seg2 RAW 段归属容器 | `L_anchor` = 锚点 CE | 干净锚点 |
| 一致性 consistency | 干净/带噪同字段表示对齐 | `L_consist` = 对比/KL(**oracle 池化教师强制**) | 配对视图 |

- 抽取头(boundary/binding/re-anchor)按预测标签**重分组 token → 产出干净结构化记录**(= B 的符号输入,见 §1.4)。
- `H_fld` 仅**内部**用于 `L_consist` 池化 + 探针,**不作 B 接口**;池化键双时态见 §2.4。

### 6.2 内容保留:已由"保留真实值"兜底(不必单列损失)

surface **保留真实值**(§2.2)→ 值形信号**天然在输入里**,抽取头指向真实值 token → 内容随抽取保留。
故 **`L_content` 作为损失基本冗余**,降级为**可选探针**(掩码字段名/值形预测),仅诊断是否 semantics-blind。

### 6.3 `c_struct`:派生量,**非训练头**

`c_struct(field) = min(段流 conf, boundary/binding/re-anchor 在该字段的预测熵)`——**零额外头**。
仅当需要**校准过**的置信(对 oracle"该字段是否真被正确恢复"回归)才升级为 `L_conf` 头(**Stage-2 / 按需**)。
用途:seg2 弃权、下游三路线的 LLM 预算分配、退化鲁棒曲线。

### 6.4 噪声类型辅助头(可选,诊断 + 增益)

- 预测施加了哪层/哪个算子(noise-meta 当标签);`L_noise` = CE。
- 帮助表示对噪声敏感,且产出可解释的退化诊断。

### 6.5 下游语义类型标注器(模块 B,**延后**,三条路线)

编码器抽取出的**干净结构化记录**进 B。三条路线(标注规模见 §九):
1. **TURL 式微调 / 轻量头**:在冻结结构表示上 few-shot 线性/浅层头(~300–750 条)。
2. **LLM zero-shot prompt**:0 训练标签,仅共用 gold eval。
3. **Projection 对齐**:结构向量投影进 LLM 嵌入(stage1 用 skeleton 自动描述、零语义标注;stage2 ~500–1500 条 typing)。

按 `c_struct` 路由:轻量头扛大头,LLM 扛开集/黑话残差。**LLM 是下游消费者,非主任务老师。**

### 6.6 总损失(预训练主干)

```
L = λ1·L_bound + λ2·L_bind + λ3·L_anchor   (抽取式结构重建:边界/字段/坏块)
  + λ4·L_consist                            (退化鲁棒 / 一致性)
  [+ 可选探针:L_content / L_noise / L_conf 校准 —— 按需,非主干]
```
下游 `L_type` **不在预训练**。多损失用不确定性加权 / GradNorm 自动平衡 `λ`,分阶段加,免网格。

### 6.7 linear-probe 监控(**锁 3**)

预训练期周期性冻结编码器,在**几百条 gold** 上跑线性 CTA probe,**仅作 sanity check**:
监测 `H_fld` 是否在学 typing 相关结构,防"结构指标刷漂亮、表示对 CTA 无用"的目标-下游错配。
零架构承诺,只需极小 gold 集。

## 九、重点内容解读

- `provenance`定义为**一个值(或它的 token)在干净结构里,真正挂在哪个字段上。** 就是"这串字符到底是谁的值"。对 token 序列的每个位置 `i`,`provenance(i) = 该 token 真正归属的字段 id`。
  - **为什么需要它**:任务是**逐字段**的(给字段定型、比较一个字段的干净版与带噪版),但编码器看到的是一串 token,**token 和字段不是天生对齐的**——尤其噪声会让它们错位。所以你必须有一张"token → 字段"的映射,这张映射就是 provenance。
  - 观测provenance:容错解析器在**带噪文本**上给出的归组。可观测,但**噪声下会错**。
  - **oracle provenance**:synth 注噪前的**干净结构**已知的真实归属。**永远正确**,但只有合成阶段有。
- `H_fld`池化：Transformer 编码器吃 token 序列 `t₁…tₙ`,吐**每 token 一个隐向量** `h₁…hₙ`(各 d 维)。但下游要的是**每字段一个向量**(给字段定型只需一个表示)。所以必须把"属于同一字段的那些 token 向量"**聚合成一个字段向量**——这步聚合就叫**池化(pooling)**。
  - **provenance 在这里的角色 = 池化的"分组键"**:它决定哪些 token 向量被归到同一个字段一起池化。`Pool` 可以是 mean / attention 加权 / max / 取该 span 的 [CLS]。
- provenance双时态表述：
  - 为什么要有双时态：
  - 要算 per-field 损失(typing、一致性),你得先有 `H_fld`;要池化出 `H_fld`,你得先有**分组键 provenance**;可 provenance 恰恰是 **field-binding 头要学着预测的东西**。于是:训练刚开始,头是随机初始化的 → 它预测的 provenance 是垃圾 → 若用这个垃圾分组去池化 → `H_fld` 池到了一堆错 token → typing/一致性损失建立在错字段上 → 梯度方向是乱的 → 头永远学不好。**循环锁死。**

- field-binding 头：学出来的provenance的生产者——**逐 token 预测它归属哪个字段槽**
  - 推理时 oracle 没了,但池化**必须有分组键**才能把 token 向量聚成 `H_fld`。这个键由谁给?——就是 field-binding 头。**没有它,预训练好的编码器在推理期根本用不起来**:你有一堆 token 向量,却不知道哪些该归一个字段 → 池不出 `H_fld` → typing 无从谈起。它就是把"token 级编码器"接到"字段级任务"的那一环。
  - `L_bind` 回传梯度,逼每个 token 的隐向量 `hᵢ` 携带"我属于哪个字段"的信息。这把表示**朝任务需要的结构感知方向**塑形,而不是只学通用去噪——是防"structure/semantics-blind"的力之一。

- 三层次噪声需要从结构上拆分，用编码器架构来学习形式层和结构层的结构鲁棒表达；用DL/LLM（TTURL 式微调或LLM zero-shot prompt /Projetcion）来做语义类型标注器。

  - 这样做的好处是：语义层噪声和结构层+形成层噪声是解耦的，而编码器对于语义的理解能力远不如LLM，且做编码器的语义理解需要大量的数据标注。

  - |                | 形式层 + 结构层                                            | 语义层                                            |
    | :------------- | :--------------------------------------------------------- | :------------------------------------------------ |
    | 腐蚀性质       | 表面/结构被破坏,但**信息仍在输入里**(分隔符冗余、结构冗余) | 名字混淆/黑话/同名异义,**意义信息根本不在输入里** |
    | 恢复方式       | **去噪 / 重建**(从输入冗余可逆)                            | **知识接地**(需外部世界知识,输入里没有)           |
    | 监督           | **oracle**(synth 已知干净结构,零语义标签)                  | **语义标签**(gold/silver,DL/LLM)                  |
    | 该用的归纳偏置 | 结构感知去噪器(自监督)                                     | 知识丰富的 LM                                     |

  - 训练标签：喂给模型、让它**调整参数**去拟合的答案。模型在训练中**看得见**它们,并直接据此更新权重。它的作用是"教"——告诉模型什么输入对应什么输出。

  - 评测标签：用来**测量**一个已经训好的模型有多准的答案。模型在评测时**只接受输入、不接受这些答案**;答案只在模型给出预测**之后**,拿来和预测对比、算指标(F1/准确率)。**训练标签和评测标签所在的数据必须严格不相交**。

  - 针对三条路线：都需要共用的评测金标，做eval-only,报 typing 性能。**不进训练**，且覆盖类型 taxonomy(假设 ~15 类:phone/email/id_card/name/address/.../非敏感),每类至少 30–50 条,且**必须跨退化档分层**(干净 + 各噪声层 + 各强度都要有),这样才能画退化曲线。算下来 **~600–1000 条字段级金标**。

  - LLM zero-shot 训练标签 = 0,因为它根本不训，但它**仍然要评测标签**——你总得知道它判得准不准。所以 zero-shot 不是"不要标签",是"**只要评测标签、不要训练标签**"。

  - 监督分类头：从头训一个分类头,要足够多的 (字段→类型) 标注对覆盖每个类、每种退化形态，**规模估算**:~15 类,监督分类要每类 ~200–500 条才训得稳,且要跨退化档 → **保守 3000–8000 条字段级标注**(训练)+ 上面的评测金标。**但有个救法把它从 8000 降到几百:不要从头训分类头,而是在你预训练好的冻结结构表示上做 few-shot 线性探针。** 因为表示已经预训练好,下游分类头只需要少量标签就能收敛:**few-shot 版规模**:每类 ~20–50 条 → **~300–750 条**即可训一个线性/浅层头。这跟评测金标同量级,甚至可以从同一批人工标注里切分(70% 训 / 30% eval)。

  - 接 LLM + 训投影层 ：投影器对齐的第二阶段(任务微调)要 typing 标签来训。规模介于 (a) few-shot 和 0 之间,因为投影器参数少、又有 LLM 先验托底 → **~500–1500 条**字段级标注。但工程重(投影器 + LoRA + 两阶段)。**对齐阶段(stage 1)其实不要语义标签**:还记得我们之前定的——投影器对齐用"结构向量→结构描述",描述从你的 skeleton **自动生成**(路径/邻域/类型,你都有),**零语义标注**。所以 B1 的标注只压在 stage 2 的 ~500–1500 条 typing 上。 半自动化来标

- **噪声分类:算子层 vs 效应层(解决"算子 × 格式"稀疏矩阵)**

  - **问题**:算子是**格式专属机制**,不能均匀套到每格式——JSON 无"列漂移"(无列),CSV 无"嵌套错位"(平)。若在**全局算子集**上均匀采样,对不同格式**良定义不了**(CSV 上"嵌套错位"概率=0 就不叫均匀)。这是唯一真出现的逻辑问题。

  - **算子 × 格式适用矩阵**(稀疏但每格式支持 5–6 个):

    | 算子 | JSON | CSV | SQL |
    | --- | :---: | :---: | :---: |
    | 截断 / 未闭合引号 / 未转义分隔符 | ✓ | ✓ | ✓ |
    | 未闭合括号 `}`/`)` | ✓ | ✗(无括号) | ✓(paren) |
    | 列漂移 | ~(标量数组) | ✓ | ✓(元组) |
    | 嵌套错位 | ✓ | ✗(平) | ✗(平) |
    | 记录边界混入 | ✓ | ✓ | ✓ |
    | 表头坍塌 | ✗(键内联) | ✓ | ~(INSERT 列清单) |

  - **解法:把分类学从"算子"上移到"结构效应"(格式无关),效应正好一一对应 4 个头**:

    | 结构效应 | 命中头 | JSON 实现 | CSV 实现 | SQL 实现 |
    | --- | --- | --- | --- | --- |
    | **A 不可恢复区** | re-anchor(+边界) | 截断/未闭合括号 | 截断/未闭合引号 | 截断/未闭合 paren |
    | **B 边界错位** | boundary(+damaged) | 未转义分隔/嵌套错位 | 未转义分隔/列漂移 | VALUES 未转义/元组漂移 |
    | **C 值错绑** | field-binding | 嵌套错位/分隔合并 | 列漂移/分隔合并 | 元组漂移/缺列 |
    | (不变性) | consistency | 全部 | 全部 | 全部 |

  - **关键**:4 个头本就活在**效应层**,不关心哪个算子造成——JSON 用嵌套错位、CSV 用列漂移,只要都产生"值错绑",都汇进同一 `field-binding` 监督 → **任务/标签层面格式统一,无逻辑问题**。

  - **修正采样**:先在**结构效应 {A, B, C}** 上均匀 → 再为(效应, 格式)挑**该格式适用**的算子。效应 A/B/C 三格式都可达 → 均匀性良定义、跨格式可比。`noise_meta` 加 `effect` 字段。

  - **格式专属算子**(`nesting_misalign`→JSON、`header_collapse`→CSV)当**该效应的格式专属额外实现**,只在该格式训练/评测。评测两档报:**per-effect(跨格式)+ per-format**。
