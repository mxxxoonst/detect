# 硕士开题相关论文调研

> 主题：面向企业真实数据湖中多源异构噪声序列数据（JSON / CSV / SQL）的「结构退化条件下的 PII 识别与关联融合」。
> 任务本质：结构退化下的字段语义类型标注（CTA 式，主）+ 值级 span 检测（NER 式，辅）。
> 本调研按三个研究方向分组，外加一组跨方向共性技术。每条均经 WebSearch 检索、WebFetch / 多源比对核实标题、venue、URL。
> 最后附「检索说明」。

---

## 方向一｜统一中间表示构建 + 结构鲁棒性感知建模

> 关注点：异构表/树/关系输入 → 统一结构化表示；代码/结构预训练编码器作基座；结构感知建模与对行列扰动的鲁棒性。

### 1. GraphCodeBERT: Pre-training Code Representations with Data Flow
- **论文题目**：GraphCodeBERT: Pre-training Code Representations with Data Flow
- **发表会议/期刊**：ICLR 2021（arXiv preprint, 2020，2009.08366）
- **研究内容**：在代码预训练中引入「数据流」这一语义级结构（变量间"值从哪来"的关系），而非语法树 AST，并设计两个结构感知预训练任务（预测结构边、对齐源码与结构表示）。直接支撑方向一以 GraphCodeBERT 为基座：它证明了把"结构图"显式编码进 Transformer 能提升结构理解，本研究可借鉴其"数据流边 + 表示对齐"思路来编码字段路径/schema 骨架/局部拓扑。
- **URL**：https://arxiv.org/abs/2009.08366

### 2. UniXcoder: Unified Cross-Modal Pre-training for Code Representation
- **论文题目**：UniXcoder: Unified Cross-Modal Pre-training for Code Representation
- **发表会议/期刊**：ACL 2022（arXiv preprint, 2022，2203.03850）
- **研究内容**：统一编码器-解码器代码预训练模型，用前缀适配器 + 掩码注意力矩阵切换理解/生成行为，并把 AST 经一对一映射序列化后与注释一起作为跨模态信息增强代码表示。作为方向一备选基座，其"把树型 AST 一对一映射为序列再喂 Transformer"的技巧可直接迁移到 JSON 树/嵌套 schema 的线性化编码，且同一模型兼顾理解与生成便于后续扩展。
- **URL**：https://arxiv.org/abs/2203.03850

### 3. TableFormer: Robust Transformer Modeling for Table-Text Encoding
- **论文题目**：TableFormer: Robust Transformer Modeling for Table-Text Encoding
- **发表会议/期刊**：ACL 2022（arXiv preprint, 2022，2203.00274）
- **研究内容**：通过可学习的注意力偏置把表格结构（行/列/表头关系）注入 Transformer，使模型对行列顺序严格不变，避免线性化时把行列序当成虚假偏置。强相关于方向一的"结构鲁棒性感知建模"：本研究的结构层噪声（列漂移、记录边界混排）正是行列顺序/位置扰动的极端形式，TableFormer 的结构性注意力偏置是抵抗这类扰动的可借鉴机制。
- **URL**：https://arxiv.org/abs/2203.00274

### 4. TUTA: Tree-based Transformers for Generally Structured Table Pre-training
- **论文题目**：TUTA: Tree-based Transformers for Generally Structured Table Pre-training
- **发表会议/期刊**：KDD 2021（arXiv preprint, 2020，2010.12537）
- **研究内容**：提出"双维坐标树"统一刻画一般结构表格的空间与层次信息，配合树based 注意力 / 位置编码与 token/cell/table 三级渐进预训练目标，覆盖关系表之外的复杂结构表。对方向一的统一中间表示（IR）有直接参考价值：其坐标树可同时表达 CSV 的二维坐标与 JSON 的嵌套层次，是"字段路径 + schema 骨架"统一编码的现成结构先验。
- **URL**：https://arxiv.org/abs/2010.12537

### 5. TaBERT: Pretraining for Joint Understanding of Textual and Tabular Data
- **论文题目**：TaBERT: Pretraining for Joint Understanding of Textual and Tabular Data
- **发表会议/期刊**：ACL 2020（arXiv preprint, 2020，2005.08314）
- **研究内容**：在 2600 万张表及其文本上联合预训练，提出"内容快照（content snapshot）"只选与上下文最相关的少量行做编码，得到 NL 与（半）结构化表的联合表示。其"代表实例值快照"思想正对应方向一 IR 中的"代表实例值"分量——在 GB 级数据下不能整表编码，TaBERT 的抽样行编码策略可直接借鉴用于构造字段级代表值表示。
- **URL**：https://arxiv.org/abs/2005.08314

### 6. TableLoRA: Low-rank Adaptation on Table Structure Understanding for Large Language Models
- **论文题目**：TableLoRA: Low-rank Adaptation on Table Structure Understanding for Large Language Models
- **发表会议/期刊**：arXiv preprint, 2025（2503.04396）（未能联网核实是否已被正式会议接收，待确认）
- **研究内容**：把 LoRA 与表格结构理解结合，引入特殊的结构感知 token 与二维 LoRA 编码方式，让 LLM 以参数高效方式获得表格结构感知能力。几乎与方向一的核心设定（结构编码 + LoRA 参数高效适配）一一对应，是最贴近的近期同类工作，可作为方法对照与差异化论证的关键 baseline。
- **URL**：https://arxiv.org/abs/2503.04396

## 方向二｜结构缺失与语义歧义下的鲁棒敏感信息识别与关联融合

> 关注点：CTA / PII 识别谱系；多证据融合；结构鲁棒的表格 QA；"结构即模态"把结构表示注入 LLM。

### 1. Sherlock: A Deep Learning Approach to Semantic Data Type Detection
- **论文题目**：Sherlock: A Deep Learning Approach to Semantic Data Type Detection
- **发表会议/期刊**：KDD 2019（arXiv preprint, 2019，1905.10688）
- **研究内容**：用多输入深度网络从列值的统计特征、字符分布、词向量、段向量等 1588 维特征预测 78 种语义类型，是 CTA 任务的奠基性深度方法。对应方向二中"基于 value 画像的候选检测"分支：本研究的 value 画像（长度/字符宏类/脚本直方图）与 Sherlock 的特征工程思路同源，可作为 s_value 通道的经典对照。
- **URL**：https://arxiv.org/abs/1905.10688

### 2. Sato: Contextual Semantic Type Detection in Tables
- **论文题目**：Sato: Contextual Semantic Type Detection in Tables
- **发表会议/期刊**：VLDB 2020 / PVLDB Vol.13（arXiv preprint, 2019，1911.06311）
- **研究内容**：在 Sherlock 的列内证据基础上，加入表级主题（LDA topic）上下文与结构化预测（CRF），让相邻列的类型相互约束。直接对应方向二的"结构上下文校正"与"多证据融合"：其"列内证据 + 表上下文 + 列间结构化预测"的组合，是 s_value 与 s_context 融合的经典范式。
- **URL**：https://arxiv.org/abs/1911.06311

### 3. TURL: Table Understanding through Representation Learning
- **论文题目**：TURL: Table Understanding through Representation Learning
- **发表会议/期刊**：VLDB 2021 / PVLDB Vol.14（arXiv preprint, 2020，2006.14806）
- **研究内容**：把"预训练-微调"范式引入关系型 Web 表，用结构感知 Transformer 编码行列结构 + 掩码实体恢复（MER）目标，统一支撑列类型标注、关系抽取、实体链接等 6 类任务。对方向二的"实体级关联融合"特别相关：其实体链接/关系抽取能力与本研究"跨记录/跨表实体关联融合"目标一致，s_cross 证据通道可借鉴其结构感知实体表示。
- **URL**：https://arxiv.org/abs/2006.14806

### 4. RECA: Related Tables Enhanced Column Semantic Type Annotation Framework
- **论文题目**：RECA: Related Tables Enhanced Column Semantic Type Annotation Framework
- **发表会议/期刊**：VLDB 2023 / PVLDB Vol.16（论文页：vldb.org/pvldb/vol16/p1319-sun.pdf）
- **研究内容**：通过命名实体 schema 找到并对齐 schema 相似、主题相关的"相关表"，把跨表上下文注入目标列做 CTA，并能处理宽表。直接对应方向二的"结构置信度感知多证据融合"中的跨表证据 s_cross：在无表头/通用化命名的语义层退化场景下，RECA 的"借助同义/相关表互证"思路正可补偿单表语义不足。
- **URL**：https://dl.acm.org/doi/10.14778/3583140.3583149

### 5. Column Type Annotation using ChatGPT
- **论文题目**：Column Type Annotation using ChatGPT
- **发表会议/期刊**：arXiv preprint, 2023（2306.00745）；相关短版见 CEUR-WS TADA 工作坊（Korini & Bizer）
- **研究内容**：系统评测 ChatGPT 在零样本/少样本下做 CTA 的多种提示设计（列级 / 文本分类 / 表级上下文）与两步流水线，F1 超 85%，用极少任务数据即媲美微调模型。对应方向二"LLM 协同"分支的提示词基线：本研究主张"结构即模态"软 token 注入而非纯提示工程，该文恰是需要对比并超越的纯提示工程 baseline。
- **URL**：https://arxiv.org/abs/2306.00745

### 6. ArcheType: A Novel Framework for Open-Source Column Type Annotation using Large Language Models
- **论文题目**：ArcheType: A Novel Framework for Open-Source Column Type Annotation using Large Language Models
- **发表会议/期刊**：VLDB 2024 / PVLDB（arXiv preprint, 2023，2310.18208）
- **研究内容**：提出上下文采样 + 提示序列化 + 模型查询 + 标签重映射的全零样本 CTA 框架，在零样本 CTA 基准上达 SOTA，且与经典方法结合后在 SOTAB 上超过微调的 DoDuo。强相关于方向二在"语义层歧义、类型集开放"场景的需求：其"标签重映射"机制解决了固定训练期类型集的局限，可借鉴用于本研究开放 PII 类型体系。
- **URL**：https://arxiv.org/abs/2310.18208

### 7. RobuT: A Systematic Study of Table QA Robustness Against Human-Annotated Adversarial Perturbations
- **论文题目**：RobuT: A Systematic Study of Table QA Robustness Against Human-Annotated Adversarial Perturbations
- **发表会议/期刊**：ACL 2023（arXiv preprint, 2023，2306.14321）
- **研究内容**：构建 13.8 万条人工对抗扰动（表头/表内容/问题三层）基准，发现 SOTA 表格 QA 模型与 GPT-3 少样本在扰动下均显著退化，并用 LLM 生成对抗样本增强训练以提升鲁棒性。对应方向二/方向一共同的鲁棒性评测设定：其"表头/内容/结构三层扰动"分层与本研究"形式/结构/语义三层噪声"分层高度同构，可借鉴其扰动构造与评测协议。
- **URL**：https://arxiv.org/abs/2306.14321

## 方向三（暂定）｜规则—中模型—大模型级联动态路由与协同优化

> 关注点：成本-性能感知的多级路由（升级/回退/终止）、级联、早退出、规则+模型混合。

### 1. FrugalGPT: How to Use Large Language Models While Reducing Cost and Improving Performance
- **论文题目**：FrugalGPT: How to Use Large Language Models While Reducing Cost and Improving Performance
- **发表会议/期刊**：arXiv preprint, 2023（2305.05176）；后见于 NeurIPS 2024 相关收录线索（正式去向待确认）
- **研究内容**：提出 LLM cascade：按查询自适应地从便宜到昂贵依次调用模型，用打分器判断是否接受当前答复，配合提示适配与 LLM 近似，可用至多 98% 成本降幅匹配 GPT-4。直接奠定方向三的级联骨架：其"逐级调用 + 置信度打分决定是否升级"正是本研究"规则→中模型→大模型"路由的核心机制原型。
- **URL**：https://arxiv.org/abs/2305.05176

### 2. RouteLLM: Learning to Route LLMs with Preference Data
- **论文题目**：RouteLLM: Learning to Route LLMs with Preference Data
- **发表会议/期刊**：arXiv preprint, 2024（2406.18665）（后续被 ICLR 2025 接收线索，正式去向待确认）
- **研究内容**：用人类偏好数据 + 数据增强训练"路由器"，在调用前就决定把查询分给强模型还是弱模型，在多基准上成本降一半以上而质量不降，且对强/弱模型替换具迁移性。对应方向三的"路由器学习"：本研究的复杂度度量 c(x)=α·n(x)+β·(1-I(x)) 可作为 RouteLLM 式路由器的输入特征，借鉴其"调用前路由"以省去无谓的高成本调用。
- **URL**：https://arxiv.org/abs/2406.18665

### 3. An Evaluation Study of Hybrid Methods for Multilingual PII Detection (RECAP)
- **论文题目**：An Evaluation Study of Hybrid Methods for Multilingual PII Detection
- **发表会议/期刊**：arXiv preprint, 2025（2510.07551）；NeurIPS 2025 相关 venue 列出（正式 track 待确认）
- **研究内容**：提出 RECAP 混合框架，把确定性正则与上下文感知 LLM 结合，三阶段精炼流水线消歧/过滤，无需重训支持 300+ 实体类型，跨 13 种低资源语言上加权 F1 大幅超过微调 NER 与零样本 LLM。同时贴合方向三与方向二：它正是"规则（正则）+ 大模型"两级协同的 PII 检测实例，是本研究"规则—中模型—大模型级联 + PII 任务"最直接的近期同类系统。
- **URL**：https://arxiv.org/abs/2510.07551

### 4. Dynamic Model Routing and Cascading for Efficient LLM Inference: A Survey
- **论文题目**：Dynamic Model Routing and Cascading for Efficient LLM Inference: A Survey
- **发表会议/期刊**：arXiv preprint, 2026（2603.04445）（综述，未能核实是否投稿正式会议，待确认）
- **研究内容**：系统梳理动态路由与级联的范式、置信度/不确定度驱动的延迟决策、调用前路由 vs 事后路由、成本感知的早丢弃/早退出等技术谱系。可作为方向三的工作脉络地图与术语对齐参考：帮助定位本研究"升级/回退/终止"三动作在既有路由分类学中的位置，规避重复造轮子。
- **URL**：https://arxiv.org/abs/2603.04445

### 5. Universal Model Routing for Efficient LLM Inference
- **论文题目**：Universal Model Routing for Efficient LLM Inference
- **发表会议/期刊**：arXiv preprint, 2025（2502.08773）（正式发表去向待确认）
- **研究内容**：研究面向"测试期才出现的新模型集合"的通用路由，用模型能力的特征化表示驱动路由，使路由策略不绑定固定模型池。对方向三的工程鲁棒性相关：企业落地常需替换/新增模型，其"路由器对模型集合变化鲁棒"特性可借鉴，避免每次换模型都重训路由器。
- **URL**：https://arxiv.org/abs/2502.08773

## 跨方向 / 共性技术：结构即模态、参数高效微调、噪声鲁棒训练

> 关注点：把非文本结构作为"模态"投影进 LLM 嵌入空间（GraphToken / LLaVA / BLIP-2 / GraphGPT）；PEFT（LoRA）；噪声鲁棒/稳定性训练（NAT）。

### 1. Let Your Graph Do the Talking: Encoding Structured Data for LLMs (GraphToken)
- **论文题目**：Let Your Graph Do the Talking: Encoding Structured Data for LLMs
- **发表会议/期刊**：arXiv preprint, 2024（2402.05862）（ICML 2024 相关收录线索，正式去向待确认）
- **研究内容**：提出 GraphToken，用参数高效方式学习一个编码函数，把结构化（图）数据编码为显式的连续 token 扩展进 LLM 提示，在 GraphQA 的节点/边/图级任务上提升最高 73 个百分点。这是方向二"结构即模态"软 token 注入的直接先例：本研究把字段级结构表示 H_fld 经 MLP 投影成软 token 注入 LLM，正是 GraphToken 思路在"字段结构"上的迁移。
- **URL**：https://arxiv.org/abs/2402.05862

### 2. Visual Instruction Tuning (LLaVA)
- **论文题目**：Visual Instruction Tuning
- **发表会议/期刊**：NeurIPS 2023（arXiv preprint, 2023，2304.08485）
- **研究内容**：首次用纯语言 GPT-4 生成多模态指令数据，训练 LLaVA——用一个简单线性投影器把视觉编码器输出对齐进 LLM 嵌入空间，端到端获得视觉-语言对话能力。是方向二"两阶段对齐 + 投影器"的范式来源：本研究的"H_fld → MLP 投影器 → LLM 软 token"两阶段对齐直接类比 LLaVA 的视觉投影器，可借鉴其对齐数据构造与两阶段训练流程。
- **URL**：https://arxiv.org/abs/2304.08485

### 3. BLIP-2: Bootstrapping Language-Image Pre-training with Frozen Image Encoders and Large Language Models
- **论文题目**：BLIP-2: Bootstrapping Language-Image Pre-training with Frozen Image Encoders and Large Language Models
- **发表会议/期刊**：ICML 2023（arXiv preprint, 2023，2301.12597）
- **研究内容**：用轻量 Q-Former（可学习 query + 两阶段：表示学习 + 生成学习）作为冻结图像编码器与冻结 LLM 之间的信息瓶颈，以极少可训练参数达到 SOTA。对方向二的投影器设计提供更强变体：相比 LLaVA 的线性投影，Q-Former 的"查询瓶颈 + 两阶段"可借鉴用于从字段结构表示中蒸馏最有用的少量软 token，控制注入长度与成本。
- **URL**：https://arxiv.org/abs/2301.12597

### 4. GraphGPT: Graph Instruction Tuning for Large Language Models
- **论文题目**：GraphGPT: Graph Instruction Tuning for Large Language Models
- **发表会议/期刊**：SIGIR 2024（arXiv preprint, 2023，2310.13023）
- **研究内容**：通过文本-图 grounding + 双阶段图指令微调 + 轻量图-文对齐投影器，把图结构知识注入 LLM，并用自监督的结构匹配任务做对齐，在监督与零样本图任务上均强泛化。是方向二"结构即模态"最新的完整工程范式：其"双阶段对齐 + 轻量投影器 + 结构匹配自监督"可整体迁移到本研究的字段结构注入，且零样本泛化对应开放 PII 类型需求。
- **URL**：https://arxiv.org/abs/2310.13023

### 5. LoRA: Low-Rank Adaptation of Large Language Models
- **论文题目**：LoRA: Low-Rank Adaptation of Large Language Models
- **发表会议/期刊**：ICLR 2022（arXiv preprint, 2021，2106.09685）
- **研究内容**：冻结预训练权重、在 Transformer 各层注入可训练的低秩分解矩阵，可训练参数减少上万倍而性能持平甚至超过全量微调。是方向一"LoRA 参数高效适配"的方法本体：本研究在 GraphCodeBERT/UniXcoder 基座上做 LoRA 适配、以及投影器对齐阶段冻结大模型，均以本工作为技术基础。
- **URL**：https://arxiv.org/abs/2106.09685

### 6. NAT: Noise-Aware Training for Robust Neural Sequence Labeling
- **论文题目**：NAT: Noise-Aware Training for Robust Neural Sequence Labeling
- **发表会议/期刊**：ACL 2020（arXiv preprint, 2020，2005.07162）
- **研究内容**：提出两类噪声感知训练目标——数据增强（干净+带噪混合训练）与稳定性训练（鼓励干净/带噪输入产生噪声不变的隐表示），在英/德 NER 上用 OCR 误差与拼写扰动验证鲁棒性提升且不损原始精度。这是方向一一致性损失 L_consist 的直接理论来源：本研究的"干净 x 与带噪 x' 对齐"正是 NAT 稳定性训练在结构噪声上的推广，可借鉴其增强 + 稳定性双目标设计。
- **URL**：https://arxiv.org/abs/2005.07162

## 与本研究的差异点（论文没做、本研究要做）

> 仅针对上表「必读」档 9 篇。每行：论文做了什么（范围）→ 它没覆盖、正是本研究要补的增量（紧扣三层噪声 / 结构退化 / c_struct 加权融合 / 结构即模态 / 成本路由）。

| 论文 | 论文的范围 | 本研究的增量 / 差异 |
| --- | --- | --- |
| TableLoRA | 在**干净、结构良好**的表格上，用结构感知 token + 2D LoRA 让 LLM 获得表结构感知，提升下游表任务。 | 它假设结构可靠，不建模**结构退化**；本研究在解析受损/schema 偏移/字段义不定的三层噪声下做适配，并输出**字段级结构置信度 c_struct** 以加权后续证据，结构本身的可信度是被显式估计而非默认为 1。 |
| GraphCodeBERT | 在源代码上用数据流边做结构预训练，得到结构感知代码表示。 | 它面向代码、且输入结构完整；本研究把它当基座迁移到 **JSON/CSV/SQL 异构数据**的统一 IR，并在**可控注噪三元组（干净 x / 带噪 x'）** 上加 L_consist 一致性损失，使表示对结构层噪声不变——这是原工作未涉及的鲁棒性训练设定。 |
| Doduo | 干净表上多列联合 CTA + 列关系预测，整表共享上下文，**微调**得到固定类型集。 | 它依赖完整表头与可靠结构，类型集训练期固定；本研究在**无表头/列漂移/语义层歧义**下做 CTA，结构证据按 c_struct 折扣，且面向**开放 PII 类型**与值级 span 检测（NER 辅），并以「结构即模态」注入而非纯整表编码。 |
| RECA | 用命名实体 schema 找「相关表」做跨表互证增强目标列 CTA。 | 它的跨表对齐**依赖可用的 schema/表头信息**来定位相关表；本研究在结构退化、表头缺失场景下仍需 s_cross，且把跨表证据纳入 **c_struct 加权的多证据融合 s=ψ(s_value, c_struct·s_schema, c_struct·s_context, s_cross)** 统一打分，而非单一增强分支。 |
| RECAP（多语言 PII） | 正则 + LLM 三阶段精炼做多语言 PII 检测，免重训支持 300+ 实体，主攻**语言多样性**。 | 它的输入是**相对规整的文本**，处理的是语言异质而非**结构退化**；本研究的级联输入是 JSON/CSV/SQL 噪声序列，路由决策由结构复杂度 + 信息缺失 **c(x)=α·n(x)+β·(1−I(x))** 驱动，且把规则—中模型—大模型做**成本感知动态路由**（升级/回退/终止），而非固定两级流水线。 |
| GraphToken | 用 PEFT 学编码函数把**图**编码为连续 token 注入 LLM 提示，提升图推理。 | 它面向通用图、且图结构本身是干净给定的；本研究把**字段级结构表示 H_fld**（来自退化数据的 IR）经投影注入 LLM，软 token 需携带 **c_struct 置信度**以表达「这段结构有多可信」，并服务 PII 类型标注而非图 QA——把「结构即模态」从干净图推广到**带噪结构**。 |
| FrugalGPT | LLM cascade：便宜→昂贵逐级调用 + 打分器决定是否接受，降成本。 | 它只在**纯 LLM 之间**级联，且面向通用 QA；本研究把**规则 / 中模型 / 大模型**三类异构求解器统一进路由，代价函数显式纳入**结构复杂度与信息缺失量 I(x)**，并支持回退与终止三动作，针对结构退化 PII 任务而非开放域问答。 |
| RouteLLM | 用偏好数据学「调用前路由器」在强/弱两模型间二选一。 | 它是**二元强/弱路由**、特征来自查询文本本身；本研究路由的是**多级异构管线**，输入特征为结构化的 c(x)=α·n(x)+β·(1−I(x))（噪声/结构退化量化），且路由结果要与前两方向的 c_struct/置信度联动，而非独立的偏好分类器。 |
| NAT | 在 NER 上用数据增强 + 稳定性训练抵抗 **OCR/拼写**等 token 级噪声。 | 它的噪声是**字符/词级文本扰动**；本研究把稳定性训练推广到**结构级噪声**（列漂移、记录边界混排、schema 偏移），干净 x 与带噪 x' 的一致性约束作用在**字段级结构表示**上，并与 c_struct 估计联合训练——是 NAT 在结构维度的扩展。 |
